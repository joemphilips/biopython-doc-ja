5.3 コードを寄贈する

生物学に関わるプログラムを Python で開発しようという気持ちがあれば， Biopython のコード開発に参加する上では何の障害もありません． メーリングリストに参加するだけで，Biopython への参加意思を表明できます - コードの開発に興味があり，どの分野の作業ができるかを教えてください． 通常，我々は新たなモジュールを開発するまえに議論を行って，アイデアを 磨こうとします - それが終わったら，あとはコーディングするだけです．

メインの Biopython リリースでは，ユーザが簡単に使えるようにするため， コードに一元性と可搬性をもたせようとしています． Biopython で使っているコーディングスタイルのガイドラインは (正式でないものですが) Biopython への参加ガイド http://www.Biopython.org/docs/developer/contrib.html で読めます． また，配布物のコードにはテスト (回帰テストフレームワークについては 4.2 を参照してください) とドキュメントを追加して， モジュールが正しく動作し，可能な限り詳細にドキュメント化するように しています．もちろん．これは理想的な場合で，多くの場合は， 自分のコードに対してテスト追加してくれたり，ドキュメントを書いて くれる人をメーリングリストで見つけられるでしょう．ですから， 好きに作業を始めてもらってかまいません．

それから，もし何らかのコードを持っていて， Biopython の 配布物に入れるのは適切でないけれども，他の人が使えるように したいと思っているなら，スクリプトセンタを (http://www.Biopython.org/scriptcentral/) を検討してみてください．ここにはバイオインフォマティクス向けの フリーの Python コードへのポインタがまとめられています．

このドキュメントを読んでくれたあなたが，Biopython を使ってみよう (そしてプロジェクトに参加してみよう) という気になってくれますように． ここまで読んでくださったあなたに感謝します！
