1.3 FAQ

  1. ある処理を行っているコードを探そうとしたのですが，コードの 入っているはずのディレクトリを見ても見つかりませんでした．どこに隠され ているんですか？

    まず， __init__.py と呼ばれるファイルにコードを書いていることを 知っておいてもらわねばなりません．普段__init__.py の中に目的のコードを 探そうとしていなければ，混乱するかもしれませんね．__init__.py の 中にコードを書くのは，ユーザが import 作業を楽にできるようにするためです． たとえば，from Bio.GenBank import GenBank のような「くどい」 import 命令を実行する代わりに， ただ from Bio import GenBank を 実行するだけですみます．

  2. 回帰テストプログラムbr_regrtest.py はどうなっているの ですか？

    回帰テストフレームワークを PyUnit を使うように変更して，改行に関する 問題も修正しました．br_regrtest.py はまだありますが，その機能の ほとんどはrun_tests.py に (コピーして) 移動しました．

  3. 回帰テストを実行中に，テストのいくつかが 
    Writing: '\012', expected: '\015'
    のような出力とともに失敗するのはなぜですか？

    そんなはずは！我々は回帰テストスイートを修正して，PyUnit を使うように する同時に改行コードの問題を解決しました．もしテストが失敗するようで あれば教えてください．
