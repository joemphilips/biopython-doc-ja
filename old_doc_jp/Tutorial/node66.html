3.11.1.1 Structure オブジェクト

Structure オブジェクトは高分子を表現するデータの階層構造の 頂点に位置しています．Structure の id はユーザが指定した 文字列になります．Structure オブジェクトには，複数の Model オブジェクトが子エンティティとして入っています． ほとんどの (全てではありませんが) 結晶構造には単一のモデルしか ない一方， NMR で決定される構造には一般的にいくつかのモデルが 入っています．結晶構造において多くの分子にゆらぎが生じている場合 にも，複数のモデルができます．

3.11.1.1.1 Structure オブジェクトを構築する

Structure オブジェクトは PDBParser オブジェクトから 生成されます: 

from Bio.PDB.PDBParser import PDBParser

p=PDBParser(PERMISSIVE=1)

structure_id="1fat"

filename="pdb1fat.ent"

s=p.get_structure(structure_id, filename)

PERMISSIVE フラグは，PDB ファイルに関するよくあるいくつかの問題 (3.11.5 参照) をパーザに無視させます (とはいえ， これによっていくつかの原子や残基が失われるかもしれないので注意して ください) ．このこのフラグを指定しない場合，パーザが PDB ファイルを 構文解析している際に問題が生じるとPDBConstructionException を送出します．

3.11.1.1.2 ヘッダ (header) とトレイラ (trailer)

get_header および get_trailer といったメソッドを 用いると，PDB ファイル中のヘッダとトレイラを (文字列からなる単純なリスト で) PDBParser オブジェクトから取り出せます．
