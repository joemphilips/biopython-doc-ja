4.2.1 回帰テストを書く

Biopython に入れるモジュールには，全てテストが (そしてドキュメントも!) ついていなければなりません．そこで，いま，仮に新しいモジュール Biospam を書いたとしましょう - 回帰テストを作成するために やらなければならない作業を以下に示します:

  1. test_Biospam.py という名前のスクリプトを書きます．
    • このスクリプトは Tests という名前のディレクトリに 入れなければなりません．
    • スクリプトはモジュールの重要な機能全てをテストしなければ なりません (もちろん，テスト項目は多ければ多いほど良いですよ!)
  2. スクリプトでテスト用のファイルが必要な場合には， Tests/Biospam ディレクトリに入れておかねばなりません．
  3. テストプログラムの出力を書き出しておき，正しく出力されているか 確かめます．これには二つの方法があります:
    1. 長い方法:
      • スクリプトを実行して，出力をファイルに書き出します． Unixマシンでは， python test_Biospam.py > test_Biospam の ように実行します．これで出力がtest_Biospam というファイルに 書き出されます．
      • test_Biospam を手作業で調べ，出力内容が正しい ことを確かめます．正しく処理が行われていて，バグがないことを確認 したら，test_Biospam ファイルを編集して，先頭の行が test_Biospam になるよう書き足します．
      • test_Biospam ファイルをTests/output ディレクトリに入れます．
    2. 簡単な方法:
      • python run_tests.py -g test_Biospam.py を実行します． 回帰テストフレームワークがうまく動作して，出力ファイルを正しい場所に 出力します．
      • 出力ファイル (Tests/output/test_Biospam のはずです) を調べ，内容が全て正しいか二度確かめます．

  4. Tests ディレクトリに移り，python run_tests.py を 実行して回帰テストを走らせます．このコマンドは全てのテストを実行し， 作成したのテストも実行される (そしてパスする!) はずです．

  5. これで終わりです! 自作のモジュール用に素晴らしいテストを 作れました．おめでとう!
