4.3.2 イベント

イベントには二つの種類: info イベントと section イベントがあります． info イベントは，データストリーム中の情報のある場所をタグ付けします． section イベントはストリーム中の部分 (section) をマークします． info イベントはデータ中の特定の行に関連づけられますが，セクション イベントはそうではありません．

セクションイベント名は必ず start_EVENTNAME および end_EVENTNAME という名前にします．EVENTNAME は イベントの名前です．

例えば， FASTA 形式の配列情報向けのスキャナでは，以下のようなイベント が生成されます: 

EVENT NAME      ORIGINAL INPUT
begin_sequence  
title           >gi|132871|sp|P19947|RL30_BACSU 50S RIBOSOMAL PROTEIN L30 (BL27
sequence        MAKLEITLKRSVIGRPEDQRVTVRTLGLKKTNQTVVHEDNAAIRGMINKVSHLVSVKEQ
end_sequence
begin_sequence
title           >gi|132679|sp|P19946|RL15_BACSU 50S RIBOSOMAL PROTEIN L15
sequence        MKLHELKPSEGSRKTRNRVGRGIGSGNGKTAGKGHKGQNARSGGGVRPGFEGGQMPLFQRLPK
sequence        RKEYAVVNLDKLNGFAEGTEVTPELLLETGVISKLNAGVKILGNGKLEKKLTVKANKFSASAK
sequence        GTAEVI
end_sequence
[...]

(見栄えをよくするために一部を削って短くしてあります)

上の例では，FASTA スキャナは title，sequence， begin_sequence，end_sequence というイベントを 生成しています．begin_sequence とend_sequence が，元の FASTA 入力データのいずれの行とも関連付けられていない ことに注意してください．このイベントは，ファイル中の別々の配列 情報を区切るために使われています．

スキャナが送信できるイベントは，それぞれのデータ形式ごとに 決まっています．
