\chapter{次は何を -- Biopython プロジェクトへの貢献}

\section{特定プラットフォーム向け配布物のメンテナンス}
\label{sec:maintain-dist}

Biopython の作者たちは，ユーザのインストール作業ができるだけ簡単に
なるようにリリースを作成しようと努力しています．そのため，Biopython
のライブラリを提供できる限り沢山のインストール形式で配布しています．
リリースを更新するたびに全てのインストール形式を作成する作業は
開発者にとって膨大になり，あまり詳しくないような形式のパッケージを
メンテナンスする必要も生じることがあります．
本節では，開発者以外の人々に各プラットフォーム向けのビルドを
メンテナンスしてもらえるように，いくつかの豆知識を提供しようと
考えています．

大雑把に言えば，各プラットフォーム向けビルドのメンテナンス作業は
かなり簡単です -- やらねばならないことは，あるリリースが作成
されるたびに，特定のシステム向けのパッケージを生成する作業だけです．
もちろん，その後パッケージをチェックして，インストールがうまくいくか
確かめる必要はあります．作成されたパッケージを主要な開発スタッフの誰かに
送れば，彼らが Web サイトにパッケージへのリンクを貼り付けてくれます．
これだけで Biopython に貢献したことになります．素晴らしい！

各プラットフォーム向けに，初めて作業を行ってみようと考えている人の
手助けになるような情報を以下に示します:

\subsection{RPM 版}

RPM はいくつかのプラットフォームで利用されているかなり有名な
パッケージ管理システムです．RPM についての解説は沢山あり，
\url{http://www.rpm.org} から入手できます．あるプラットフォーム
向けの RPM の作成は非常に簡単です．必要なのは，単にソースコードから
ビルドを行える (もちろん，きちんと動作するコンパイラを持っていること
が必須条件です) ということだけです．詳しくは Biopython インストール
ガイド (Biopython installation instructions) を参照してください．

通常， RPM の作成は，以下のコマンドを実行するだけでできます:

\begin{verbatim}
python setup.py bdist_rpm
\end{verbatim}

このコマンドで，現在使っているプラットフォーム向けの RPM が
\file{dist} ディレクトリの下に作成されます．作成された RPM を問題なく
利用できれば，作業は終わりです．簡単ですね．


\subsection{Windows 版}

Windows 版には通常，よくできた GUI ベースのインストーラがついて来て，
必要なコンポーネントを正しい場所にインストールしてくれます．
この手のインストーラは，\module{Distutils} を使ってかなり簡単に
作成できます．

まず，自分の Windows コンピュータで動き，C コンパイラを持っており，
そのコンパイラで Biopython をコンパイルしてインストールできねば
なりません．(やり方は Biopython インストールガイドを参照してください)．

C コンパイラを設定してあれば，インストーラの作成は以下のように
するだけです:

\begin{verbatim}
python setup.py bdist_wininst
\end{verbatim}

これで Windows 版インストーラが手に入ります．おめでとう!

\subsection{Macintosh 版}

われわれは， Macintosh 版の配布物をメンテナンスして， bin-hex の
ような Macintosh フレンドリな形式で利用できるようにしてくれる人が
現れてくれないかと熱望しています．
基本的には， Mac 上で全てをコンパイルする方法を見つけたり，
UNIX ベースの開発者が書いたコード全てが Mac でも正しく動作することを
確認したり，Mac フレンドリなヒントを教えてくれる人が必要です．

もしあなたが Mac ユーザで，Biopython のパッケージを作成できたら，
自分のシステムでテストして，インストールが全てうまくいき，正しく
動いていそうか確認してください．うまく行っていれば Biopython 
開発者の誰かに送ってください (誰に送ればよいかわからなければ，
Biopytho@Biopython.org のメーリングリストに送ってください)
これで完了です．ありがとう！


\section{バグ報告や機能追加の要望}

Biopython モジュールに対するフィードバックは，我々開発者にとって
非常に重要な情報です．この手のオープンソースプロジェクトでは，
様々なコントリビュータからのフィードバックやバグ報告 (もちろん
パッチも!) に多大な恩恵を得ています．

機能追加のリクエストや，報告されたバグについて議論するメインの
場は，Biopython 開発者メーリングリストです:

\begin{description}
\item [\email{Biopython@Biopython.org}] -- Biopython 関係の一般的な
話題について議論するための，投稿制限のないメーリングリストです．

\item [\email{Biopython-dev@Biopython.org}] -- 開発寄りの
メーリングリストで，主に開発者が購読しています (が，誰でも参加できます)．
\end{description}

メーリングリストの他にも，もしバグと思しき症状を見つけたら，
\url{http://bugzilla.open-bio.org/} にあるバグ追跡システムに
提出できます．こちらを使えば，誰かのメールボックスに埋もれて
忘れ去られたりはしないはずです．

\section{コードを寄贈する}

生物学に関わるプログラムを Python で開発しようという気持ちがあれば，
Biopython のコード開発に参加する上では何の障害もありません．
メーリングリストに参加するだけで，Biopython への参加意思を表明できます
-- コードの開発に興味があり，どの分野の作業ができるかを教えてください．
通常，我々は新たなモジュールを開発するまえに議論を行って，アイデアを
磨こうとします -- それが終わったら，あとはコーディングするだけです．

メインの Biopython リリースでは，ユーザが簡単に使えるようにするため，
コードに一元性と可搬性をもたせようとしています．
Biopython で使っているコーディングスタイルのガイドラインは
(正式でないものですが) Biopython への参加ガイド
\url{http://www.Biopython.org/docs/developer/contrib.html} 
で読めます．
また，配布物のコードにはテスト (回帰テストフレームワークについては 
\ref{sec:regr-test} を参照してください) とドキュメントを追加して，
モジュールが正しく動作し，可能な限り詳細にドキュメント化するように
しています．もちろん．これは理想的な場合で，多くの場合は，
自分のコードに対してテスト追加してくれたり，ドキュメントを書いて
くれる人をメーリングリストで見つけられるでしょう．ですから，
好きに作業を始めてもらってかまいません．

それから，もし何らかのコードを持っていて， Biopython の
配布物に入れるのは適切でないけれども，他の人が使えるように
したいと思っているなら，スクリプトセンタを
(\url{http://www.Biopython.org/scriptcentral/}) 
を検討してみてください．ここにはバイオインフォマティクス向けの
フリーの Python コードへのポインタがまとめられています．

このドキュメントを読んでくれたあなたが，Biopython を使ってみよう
(そしてプロジェクトに参加してみよう) という気になってくれますように．
ここまで読んでくださったあなたに感謝します！
